# PRs — registro

Guarda aquí el “card” de cada Pull Request para tener trazabilidad fuera de GitHub.

## Regla (obligatoria)

Por cada PR:
1) Crea un archivo markdown en `docs/pr/YYYY/MM/`.
2) Incluye:
   - Link del PR (o compare link si aún no existe)
   - Título
   - Resumen (qué cambia)
   - Checklist de QA mínimo
   - Notas de riesgo / rollback
   - Bitácoras relacionadas

## Naming recomendado

`pr-<scope>-<branch>-<id>.md`

Ejemplos:
- `pr-search-v2-s5-chore-search-v2-s5-cleanup-040.md`
- `pr-profile-auth-hotfix-012.md`
