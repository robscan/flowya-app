# Docs — FLOWYA (memoria operativa)

Este folder está diseñado para que, con **solo estos documentos**, un consultor/IA pueda responder:
- ¿En dónde estamos?
- ¿Qué decisiones ya están cerradas?
- ¿Qué falta para cerrar el scope actual?
- ¿Qué sigue después?

## Empieza aquí

1) `docs/ops/CURRENT_STATE.md`  
2) `docs/ops/OPEN_LOOPS.md`  
3) `docs/ops/DECISIONS.md`  
4) `docs/bitacora/INDEX.md` (índices)  
5) `docs/definitions/` (contratos y reglas estables)

## Reglas

- **1 fuente de verdad por tema** (sin duplicados).
- Todo PR debe tener un “card” en `docs/pr/` (link + contexto + QA).
- La bitácora es evidencia; `ops/` es estado actual.
