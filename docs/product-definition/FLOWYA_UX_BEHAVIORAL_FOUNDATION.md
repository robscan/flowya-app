# 1. 🎯 PROPÓSITO DEL DOCUMENTO

Este documento define:

- Usuario objetivo (real, no aspiracional)
- Jobs To Be Done (JTBD) por nivel
- Modelo mental del viajero
- Principios UX/UI
- Marco de Behavioral Design (uso controlado)
- Fundamentos cognitivos (Laws of UX)

👉 Objetivo: asegurar que TODA decisión de producto esté fundamentada.

---

# 2. 🧠 POSICIONAMIENTO CLAVE

FLOWYA NO es:

- Google Maps
- Tripadvisor
- Booking

FLOWYA ES:

> Un sistema para descubrir, planear y recordar viajes con intención, contexto y emoción.

---

# 3. 🌍 INVESTIGACIÓN — TIPOS DE VIAJEROS

## A. Turista masivo ❌
- Busca eficiencia
- Sigue listas
- No es target

## B. Viajero funcional ⚠️
- Usa mapas utilitarios
- No es core

## C. Viajero experiencial ✅ (CORE)

- Busca identidad del lugar
- Valora estética y narrativa
- Prefiere descubrir

---

# 4. 👤 USER PERSONAS

## Explorador curioso (CORE)
- Viaja 4–8 veces al año
- Descubre caminando
- Quiere contexto + libertad

## Planner relajado
- Planea pero quiere flexibilidad

## Documentador
- Quiere memoria significativa

---

# 5. 🎯 JTBD POR NIVEL

## Nivel 3 — Ejecución

# 8. 🧠 BEHAVIORAL DESIGN (USO CORRECTO)
